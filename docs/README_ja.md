
# y4-clipboardMN

**Wayland統合型クリップボード・インフラストラクチャ**

`y4-clipboardMN` は、Wayland環境向けに設計された高性能かつスタンドアロンのクリップボードマネージャーです。監視（Ingress）、提供（Egress）、および永続化（Persistence）を単一のバイナリに統合し、断片化されたツールチェーンに起因する不安定性を排除します。

「絶対的な整合性」と「リソース効率」に焦点を当てて構築されており、数バイトのテキストから、70MBを超える巨大なロスレス画像まで、ゼロ・レイテンシのレスポンスで処理します。

---

## 設計思想

### 1. ライフサイクルの一元管理
監視者（Monitor）と供給者（Provider）を単一のデーモンプロセスに統合することで、同期のズレやゾンビプロセスの発生を構造的に解決しました。プロセス間通信は、Unixドメインソケットを介した厳格なIPCモデルによって制御されます。

### 2. 大容量データへの耐性
極限の負荷に耐える設計を採用。ページアライメント（4KB境界）されたメモリバッファと、シングルパスでのSHA3-256ハッシュ計算により、メモリフットプリントを最小限に抑えつつ、カーネル空間に近い速度で巨大なバイナリデータを処理します。

### 3. 鉄壁の永続化
SQLite 3（WALモード）をバックエンドに採用。ハイブリッドストレージ戦略により、メタデータとテキストは高速なインデックス検索が可能なデータベースへ、巨大なバイナリ資産は重複排除された専用のファイルシステムキャッシュ（`~/.cache/y4-clipboard/`）へと自動的に振り分けられます。

---

## 主な機能

- **Unified Daemon**: すべてのクリップボード操作を一括管理する単一のデーモン。
- **Hybrid Storage**: SQLiteによるメタデータ管理と、ファイルシステムによるバイナリキャッシュの融合。
- **Stable ID System**: 表示順に依存しない不変のデータベースID。RofiやFzf等の外部スクリプトから、レースコンディションを排した正確なアイテム特定が可能。
- **Strict CLI**: 不正な入力や未定義のフラグを冷徹に拒絶する、厳格な引数解析エンジン。
- **Security Focused**: ディレクトリ権限（700）およびファイル権限（600）の強制、機密MIMEタイプのフィルタリング。

## クイックスタート

### 1. ソースからのビルド
```bash
cargo build --release
sudo cp target/release/y4-clipboard /usr/local/bin/
```

### 2. デーモンの起動
監視およびIPCリスナーを初期化します。
```bash
y4-clipboard daemon
```

### 3. 基本操作
```bash
y4-clipboard list 0-50 --id    # 不変IDを使用して履歴を表示
y4-clipboard copy-to --id 42   # IPC経由で特定のアイテムを復元
```

## コマンドリファレンス

| コマンド | 説明 |
| :--- | :--- |
| `daemon` | バックグラウンド監視およびIPCソケットリスナーを開始。 |
| `list` | 履歴のメタデータを表示。範囲指定、Raw出力、不変ID表示に対応。 |
| `copy-to` | 指定したレコードをクリップボードに復元。MRU（最新浮上）ロジックを適用。 |
| `search` | SQLiteインデックスを利用した、テキスト履歴の高速キーワード検索。 |
| `show` | レコードの詳細を確認。`--raw` フラグでバイナリの直接抽出が可能。 |
| `store` | 標準入力からデータを吸い出し、DB保存と同時にクリップボードへ同期。 |
| `paste-from` | DBを介さず、現在のシステムクリップボードに直接アクセスして出力。 |
| `delete` | 特定のレコードを永続ストレージから物理的に削除。 |
| `wipe` | 全履歴を消去し、SQLiteのVACUUMを実行してストレージを最適化。 |

## 技術仕様

- **Language**: Rust (Zero-cost abstractions)
- **Storage**: SQLite 3 (WAL mode, Memory-mapped I/O)
- **Hashing**: SHA3-256 (FIPS 202)
- **Protocol**: Wayland `ext-data-control-v1`
- **Memory**: Page-aligned I/O, `malloc_trim` による積極的な領域解放
